Core Features

核心功能

覆盖电商订单全生命周期管理的完整能力

创建订单

接收用户ID、商品ID和购买数量，自动校验商品存在性、库存充足性，计算总价，原子性扣减库存并生成订单记录。全部步骤在事务中完成。

订单状态流转

完整的订单生命周期：待支付 → 已支付 → 已完成，以及取消操作。取消时自动归还库存（事务保证），已支付/已完成订单不可取消。

多表关联查询

通过 MyBatis resultMap + SQL JOIN 实现订单详情查询，一次请求返回订单基本信息、关联的用户姓名和商品名称。

并发安全

使用乐观锁策略（UPDATE ... WHERE stock >= #{quantity}）防止超卖，数据库行锁保证原子性，高并发下安全可靠。

全局异常处理

@ControllerAdvice 统一捕获 BusinessException、参数校验异常、绑定异常等，返回友好的 JSON 错误信息，杜绝异常堆栈泄露。

分页查询

通用 PageRequest/PageResult 分页组件，商品列表和订单列表均支持分页返回，避免全量数据传输带来的性能问题。

RESTful API

接口文档

遵循 RESTful 规范设计的完整 CRUD 接口

订单模块 /api/orders

方法路径说明
POST/api/orders创建订单（事务控制）
GET/api/orders/{id}查询订单详情（多表 JOIN）
GET/api/orders/user/{userId}按用户查询订单（分页）
PUT/api/orders/{id}/cancel取消订单（归还库存）
PUT/api/orders/{id}/pay支付订单

商品模块 /api/products

方法路径说明
GET/api/products商品列表（分页）
GET/api/products/{id}商品详情
POST/api/products新增商品
PUT/api/products/{id}修改商品
DELETE/api/products/{id}删除商品

用户模块 /api/users

方法路径说明
GET/api/users用户列表
GET/api/users/{id}用户详情
POST/api/users注册用户
PUT/api/users/{id}修改用户
DELETE/api/users/{id}删除用户

Database Design

数据库设计

三表关联：user / product / orders

user 用户表
idPKINT AUTO_INCREMENT
usernameVARCHAR(50)
phoneVARCHAR(20)
product 商品表
idPKINT AUTO_INCREMENT
product_nameVARCHAR(100)
priceDECIMAL(10,2)
stockINT DEFAULT 0
orders 订单表
idPKINT AUTO_INCREMENT
user_idFKINT
product_idFKINT
quantityINT
total_priceDECIMAL(10,2)
statusINT (0/1/2/3)
order_timeDATETIME

Quick Start

快速开始

三步启动项目

Terminal 
# 1. 克隆项目
git clone https://github.com/w020316/order-management.git
cd order-management

# 2. 初始化数据库
mysql -u root -p < sql/init.sql

# 3. 配置数据库连接 (application.yml)
# 设置 MYSQL_PASSWORD 环境变量或直接修改 yml

# 5. 访问 Swagger 文档
# http://localhost:8080/swagger-ui/index.html
